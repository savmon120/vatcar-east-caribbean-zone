# Nav

Documentation and examples for how to use Bootstrap’s included navigation components.

## Bootstrap Documentation

https://getbootstrap.com/docs/5.3/components/navs-tabs/

## Key Features

- **Flexible Alignment**: Supports multiple alignment options including left, right, center, and vertical.
- **Style Variations**: Offers styles such as tabs and pills for different visual appearances.
- **Fill Options**: Includes fill and justify options to control the distribution of navigation items.
- **Optional Dropdowns**: Expanded items render as Bootstrap dropdowns by default, or as inline nested lists when `is_dropdown` is `false` (ideal for sidebars and secondary navigation).
- **Utility Class Integration**: Allows for the addition of custom utility classes for further customization.

## Customization Options

- **Alignment Classes**: Adjust the alignment of the nav items based on the provided configuration.
- **Style and Fill**: Customize the nav's appearance by choosing between tabs or pills styles and fill options.
- **Utility Classes**: Apply additional utility classes to enhance or modify the styling and functionality of the nav component.

## Template Structure

- **Dynamic Class Application**: Automatically applies classes based on the specified alignment, style, and fill settings.
- **Item Rendering**: Iterates over navigation items, rendering each with appropriate classes based on its state (active, expanded, etc.).
- **Dropdown Support**: Includes logic for rendering dropdown menus for items with sub-navigation.

## Usage

```twig
{%
  include 'radix:nav' with {
    alignment: 'right',
    style: 'pills',
    fill: 'justify',
    items: links,
    container: 'fixed',
    color: 'light',
    navbar_utility_classes: ['bg-light'],
    nav_link_utility_classes: ['text-dark'],
    nav_item_utility_classes: ['px-2'],
  }
%}
```

### Non-dropdown (sidebar / vertical) navigation

Set `is_dropdown: false` to render expanded parent items with their children
inline as a nested vertical list instead of a Bootstrap dropdown. This is the
recommended pattern for secondary menus and side navigation.

```twig
{%
  include 'radix:nav' with {
    alignment: 'vertical',
    is_dropdown: false,
    items: links,
  }
%}
```

> **Note:** Children are only rendered when the menu item is expanded. For a
> static sidebar that should always show its child links, enable **"Expand all
> menu items"** on the menu block placement (`expand_all_items`). The
> non-dropdown list recurses, so menus deeper than two levels render in full.

### Horizontal (side-opening) dropdowns

For multi-level dropdown menus, set `dropdown_direction` to open nested
submenus to the side instead of below. Use `dropend` to open to the right or
`dropstart` to open to the left. The direction propagates to every nesting
level. Leave it empty (default) for submenus that open below.

```twig
{%
  include 'radix:nav' with {
    items: links,
    dropdown_direction: 'dropend',
  }
%}
```
