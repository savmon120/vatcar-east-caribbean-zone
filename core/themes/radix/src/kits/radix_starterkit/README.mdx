# Installation

radix_starterkit theme uses [esbuild](https://esbuild.github.io) to compile and
bundle SASS and JS.

Available scripts:

- `npm run build`: one-off production build (minified, no source maps).
- `npm run dev`: one-off development build (unminified, with source maps).
- `npm run watch`: development build that watches for changes and reloads the
  browser via BrowserSync.

#### Step 1

Make sure you have Node and npm installed.
You can read a guide on how to install node here:
https://docs.npmjs.com/getting-started/installing-node

If you prefer to use [Yarn](https://yarnpkg.com) instead of npm, install Yarn by
following the guide [here](https://yarnpkg.com/docs/install).

#### Step 2

Go to the root of radix_starterkit theme and run the following commands:
`nvm use`

This command triggers the node version switch specified in the `.nvmrc` file
Read more about [NVM](https://github.com/nvm-sh/nvm)

then:
`npm install` or `yarn install`

#### Step 3

Create a copy of **.env.example** file and rename it to **.env.local**:
`cp .env.example .env.local`

Update `DRUPAL_BASE_URL` in **.env.local** file.

> **Using DDEV?** When `npm run watch` runs *inside* the DDEV web container
> (e.g. `ddev exec npm run watch`), the build auto-detects it via the
> `IS_DDEV_PROJECT` environment variable, proxies BrowserSync to `localhost`,
> and skips opening a browser. `DRUPAL_BASE_URL` is only used when running on
> the host (outside the container).

#### Step 4

Enable and set the newly created theme as default theme.
`drush en radix_starterkit -y`
`drush config-set system.theme default radix_starterkit -y`

#### Step 5

Run the following command to compile Sass and watch for changes: `npm run watch`
or respectively `yarn watch`.

# Icons (Icon API)

This theme ships a Bootstrap Icons pack wired into Drupal's core
[Icon API](https://www.drupal.org/docs/develop/drupal-apis/icon-api), declared in
`radix_starterkit.icons.yml`.

The SVG files are not committed. They are pulled from the `bootstrap-icons` npm
dependency and copied into `assets/icons/bootstrap-icons` by the build, so run
`npm install && npm run dev` (or `yarn && yarn dev`) once before using icons.

> **First run:** the Icon API caches the discovered SVG list. If you install and
> enable this theme *before* the build has populated `assets/icons/bootstrap-icons`,
> icons render blank. Run a cache rebuild (`drush cr`) once after the first build.

Use icons in a Twig template:

```twig
{{ icon('bootstrap_icons', 'house-fill', { size: 48, color: '#0d6efd' }) }}
```

…or in a render array:

```php
$build['icon'] = [
  '#type' => 'icon',
  '#pack_id' => 'bootstrap_icons',
  '#icon_id' => 'house-fill',
  '#settings' => ['size' => 48],
];
```

Browse available icon names at https://icons.getbootstrap.com. To use a
different icon set, swap the npm dependency and update the `sources`, `template`,
and copy step accordingly.
